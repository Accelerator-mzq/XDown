# 编码（MiniMax 模型）

> **路径规则**：所有文档路径以 `smartroute.config.json` 中的 `documents` 配置为准，不要使用硬编码路径。

请根据详细设计文档开始编写代码。$ARGUMENTS

## 第一步：读取配置
1. 读取 `smartroute.config.json` 中 `documents.detailed_design` 确定详细设计文档路径

## 第二步：编码流程
1. 读取详细设计文档，确认待编码的模块和接口
2. 按模块优先级顺序编码（基础模块 → 依赖模块 → 上层模块）
3. 每完成一个模块的编码后立即编译，确保编译通过再进入下一个模块

## 编码规范
1. **语言限制**: 所有代码注释、变量解释、逻辑说明，必须 100% 使用简体中文
2. 每个函数上方必须有固定格式的函数头注释：
   ```cpp
   /******************************************************
    * @brief [中文一句话简述函数的作用]
    * @param [参数名] [中文说明参数含义]
    * @return [返回值说明]
    * @note [特别注意：例如此函数是否线程安全，或是否涉及 Qt::QueuedConnection]
    ******************************************************/
   ```
3. **逻辑注释**: 在复杂的指针操作、SQLite 事务或下载状态流转处，必须加上 `//` 中文单行注释解释原因
4. 每次修改后确保编译通过
5. 遵循详细设计中的类结构和接口定义，不得随意修改公共接口签名

## 代码组织
- 源代码存放在 `src/` 目录
- 头文件存放在 `include/` 目录（如适用）
- 文件命名与详细设计中的类名保持一致

## 编译验证
- 每个模块编写完成后执行编译命令验证
- 如果编译失败，立即修复后再继续
- 所有模块完成后执行一次完整编译，确认无错误无警告
