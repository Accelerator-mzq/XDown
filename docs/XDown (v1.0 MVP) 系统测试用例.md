###  系统测试用例 (System Test Cases)

**责任人：QA 端 (用户体验把关)** **测试方式：手工黑盒测试 (运行 appXDown.exe)** **核心目标：验证 C++ 后端与 QML 前端的协同工作，以及真实的物理环境（磁盘、网络）交互。**

|**用例编号**|**业务模块**|**测试场景**|**操作步骤**|**预期验收结果 (Acceptance Criteria)**|
|---|---|---|---|---|
|**ST-1.1**|**端到端下载**|真实网络小文件下载|1. 启动程序，点击新建任务。<br>2. 粘贴有效测试直链，确认。<br>3. 等待进度条走到 100%。|1. 列表进度条和速度实时刷新。<br>2. 状态变更为“已完成”。<br>3. 本地目标文件夹生成对应文件，MD5/Hash 校验无误。|
|**ST-1.2**|**断点续传**|支持 HTTP 206 的真实服务器|1. 下载至 30% 时点击暂停。<br>2. 强杀软件进程 (`appXDown.exe`)。<br>3. 重启软件，找到该任务，点击继续。|1. 重启后任务停留在 30%，状态为“已暂停”。<br>2. 继续后从 30% 开始增长，不断联，不从头下载。|
|**ST-1.3**|**UI 与多线程隔离**|大文件满速下载时的 UI 响应|1. 开始一个网速拉满的下载任务。<br>2. 用鼠标疯狂拖动程序窗口，高频点击左侧 Tab 切换。|UI 界面丝滑流畅（保持 60帧），没有任何拖拉卡顿、白屏或“未响应”提示。|
|**ST-2.1**|**系统级防护**|无网络环境下的错误捕获|1. 物理断开电脑网络。<br>2. 点击新建任务并尝试下载。|1. 任务不会卡死，状态变更为“错误”。<br>2. 列表上能看到对应的错误文本（如“网络连接超时”）。|
|**ST-2.2**|**内存红线测试**|极限资源监控|1. 开启 3 个并发任务全速下载。<br>2. 打开 Windows 任务管理器。|观察 `appXDown.exe` 的内存占用，无论下载多久，峰值严格锁定在 50MB 左右，无持续泄露。|
|**ST-3.1**|**大文件下载**|超大文件完整性|1. 启动程序，新建任务。<br>2. 粘贴 1GB 以上的测试直链。<br>3. 等待下载完成。|1. 下载完成无报错。<br>2. 文件 MD5/Hash 校验与服务器一致。|
|**ST-3.2**|**多文件并发下载**|超过最大并发数|1. 连续添加 5 个下载任务。<br>2. 观察下载队列表现。|前 3 个任务并行下载，后 2 个任务进入等待队列。|
|**ST-3.3**|**下载速度限制**|限速功能|1. 在设置中开启限速（如 500KB/s）。<br>2. 开始下载。|实际下载速度不超过设定值。|
|**ST-4.1**|**下载控制-停止**|取消下载|1. 开始下载任务。<br>2. 点击停止按钮。|1. 任务状态变为"已停止"。<br>2. 任务从活跃列表移除。|
|**ST-4.2**|**下载控制-删除**|删除任务|1. 创建一个已完成的任务。<br>2. 右键删除任务，勾选"同时删除本地文件"。|1. 任务从列表中移除。<br>2. 本地文件也被删除。|
|**ST-4.3**|**下载控制-删除**|仅删除任务记录|1. 创建一个已完成的任务。<br>2. 右键删除任务，不勾选"同时删除本地文件"。|1. 任务从列表中移除。<br>2. 本地文件保留。|
|**ST-5.1**|**错误处理-无效URL**|404 错误|1. 新建任务，粘贴一个不存在的 URL（如 `http://test.com/notexist.zip`）。<br>2. 确认开始下载。|1. 任务状态显示"错误"。<br>2. 显示"404 Not Found"或类似错误信息。|
|**ST-5.2**|**错误处理-网络超时**|连接超时|1. 设置较短的超时时间。<br>2. 尝试连接一个慢速响应的服务器。|任务状态显示"错误"，提示"连接超时"。|
|**ST-5.3**|**错误处理-磁盘空间不足**|磁盘满|1. 目标磁盘剩余空间小于待下载文件大小。<br>2. 开始下载。|1. 任务报错"磁盘空间不足"。<br>2. 程序不崩溃。|
|**ST-5.4**|**错误处理-写入权限**|无写入权限|1. 目标目录设为只读。<br>2. 尝试下载。|任务报错"无法写入文件"或"权限不足"。|
|**ST-5.5**|**错误处理-服务器错误**|5xx 错误|1. 尝试下载返回 500 错误的 URL。<br>2. 观察错误提示。|任务状态显示"错误"，提示"服务器内部错误"。|
|**ST-6.1**|**文件管理-目录配置**|修改默认下载路径|1. 打开设置，修改下载目录为自定义路径（如 D:\Downloads\XDown）。<br>2. 新建任务下载。|文件保存到自定义目录，而非默认目录。|
|**ST-6.2**|**文件管理-重名处理**|同名文件覆盖|1. 目标目录已存在同名文件。<br>2. 开始下载同名文件。|1. 提示文件已存在。<br>2. 自动重命名（如 file(1).zip）或提示用户选择。|
|**ST-6.3**|**文件管理-特殊字符**|特殊文件名|1. 下载文件名包含中文、空格、特殊符号（如 `测试 文件(1).zip`）。<br>2. 等待下载完成。|文件正常保存，文件名显示正确。|
|**ST-7.1**|**任务详情-信息查看**|查看任务属性|1. 右键点击已完成的任务。<br>2. 选择"查看详情"或"属性"。|显示完整的任务信息：URL、保存路径、文件大小、下载时间、速度等。|
|**ST-8.1**|**边界情况-0字节文件**|0字节文件下载|1. 尝试下载一个 0 字节的文件。|1. 任务状态正常显示"已完成"。<br>2. 不报错。|
|**ST-8.2**|**边界情况-任务重名**|重复添加相同URL|1. 连续两次添加相同的 URL 下载。|1. 第二次添加时提示任务已存在。<br>2. 或允许添加并自动重命名。|
