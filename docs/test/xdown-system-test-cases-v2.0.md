## 测试用例说明

本文档包含 XDown v2.0 的 33 项验收标准对应的详细测试用例，按功能模块分组：

- **基础功能测试**（7 个用例）：继承自 v1.0 的核心功能验证
- **多线程下载测试**（5 个用例）：验证多线程分块下载功能
- **P2P 下载测试**（6 个用例）：验证 BT 种子和磁力链接下载功能
- **界面与交互测试**（5 个用例）：验证新增的 UI 元素和交互功能
- **高级功能测试**（5 个用例）：验证自适应线程数、做种策略、完整性校验等高级功能
- **性能与稳定性测试**（5 个用例）：验证并发性能、内存占用、异常处理等

**测试状态说明**：
- ✅ Pass：测试通过
- ❌ Fail：测试失败
- ⏸️ Blocked：测试被阻塞（依赖未完成）
- ⏭️ Skipped：测试跳过
- 🔄 Pending：待测试

### TC-01：软件启动与稳定性测试

**用例编号**：TC-01
**用例名称**：软件启动与稳定性测试
**测试目标**：验证软件能够成功编译并启动，无崩溃闪退
**对应验收标准**：AC-1

**前置条件**：
- 已完成软件编译（Debug 或 Release 模式）
- 目标系统满足最低硬件要求（Windows 10/11 64-bit，8GB RAM）

**测试步骤**：
1. 双击运行 `appXDown.exe`
2. 观察软件启动过程（加载时间、界面渲染）
3. 检查主窗口是否正常显示（侧边栏、顶部操作栏、主内容区）
4. 检查是否有任何错误提示或崩溃
5. 保持软件运行 5 分钟，观察是否有内存泄漏或异常退出

**预期结果**：
1. 软件在 3 秒内成功启动
2. 主窗口完整显示，无白屏或渲染错误
3. 无任何错误提示或崩溃
4. 软件稳定运行，无内存泄漏（内存占用稳定在 50MB 以内）

3. 多线程下载时间缩短至少 50%
4. 多线程下载速度提升至少 3 倍
