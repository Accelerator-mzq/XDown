## 1. 产品概述 (Product Overview)

- **产品定位**：XDown 是一款主打极简、无广告、现代化的本地纯粹下载工具。

- **本期目标 (v1.0)**：构建前端 GUI 与后端引擎的通信桥梁，实现标准的 HTTP/HTTPS 单线程流式下载，并通过 SQLite 实现任务的持久化存储。

- **目标用户**：对下载工具有基本诉求，厌恶臃肿功能及广告的桌面用户。

---

## 2. 界面与布局规范 (UI & Layout Specifications)

v1.0 将摒弃所有复杂层级，采用经典的"双栏式"布局。

|**区域**|**包含组件**|**交互说明**|
|---|---|---|
|**侧边导航栏 (左侧)**|"下载中" Tab、"已完成" Tab|点击切换右侧主视图的数据源。选中状态需有明显的背景色或高亮视觉提示。|
|**顶部操作栏 (顶部)**|"+ 新建任务" 按钮、窗口控制键|窗口控制键支持标准的最小化、最大化、关闭功能。|
|**主内容区 (右侧)**|任务列表 (`ListView`)|基于选择的侧边栏 Tab 渲染不同的任务卡片。支持滚动条。|
|**弹窗 (模态)**|新建任务对话框|包含 URL 输入框、保存路径选择器、以及"确认/取消"按钮。|

## 3. 核心功能需求 (Functional Requirements)

### 3.1 任务创建模块 (Task Creation)

- **URL 解析**：仅支持标准 `http://` 和 `https://` 协议的链接解析。

- **路径选择**：用户新建任务时，系统需提供默认保存路径（如 Windows 的 `~/Downloads`），并允许用户调用系统原生文件管理器更改路径。

- **重复性校验**：新建任务时，需校验当前下载列表中是否存在相同 URL 且处于"下载中"的任务，若有则阻断创建并提示用户。

- **文件名自动解析**：从 URL 中自动提取文件名，若 URL 无文件名则使用默认名称。

### 3.2 核心下载引擎 (Download Engine)

- **单线程传输**：使用 `QNetworkAccessManager` 建立稳定的单 Socket 连接，获取文件字节流。

- **本地写入**：将获取的字节流持续追加写入本地文件系统（使用 64KB 固定缓冲区循环读取，避免内存溢出）。

- **断点续传**：通过 HTTP Range 请求头实现断点续传，检测服务器返回的 206 状态码判断是否支持。若服务器返回 200，则删除已下载部分重新开始。

- **HTTP 重定向**：支持 301/302/307/308 重定向，自动跟踪重定向并更新 URL（最多 10 次）。

- **异常处理**：遇到 404 (找不到资源)、403 (无权限)、连接超时等网络错误时，任务状态自动变更为"错误"，并保存错误抛出信息。

- **磁盘空间检查**：下载前检查目标磁盘剩余空间，保留 100MB 缓冲空间。

### 3.3 任务控制与状态流转 (Task Control)

- **状态定义**：任务在其生命周期内仅包含以下状态：等待中、下载中、已暂停、已完成、错误。

- **并发控制**：最多同时进行 3 个下载任务，超过限制的任务进入"等待中"队列。

- **用户指令**：

  - **开始/继续**：触发网络连接，进入"下载中"。

  - **暂停**：主动断开网络连接，保存已下载的临时文件，进入"已暂停"。

  - **删除**：从任务列表中移除记录。需弹窗二次确认："是否同时删除本地文件？"

### 3.4 任务列表展示 (List Display)

- **数据绑定**：通过 `QAbstractListModel` 实现底层数据到 QML 界面的映射。

- **卡片元素**：每个任务卡片必须实时展示以下信息：

  - 文件名 (File Name)

  - 当前下载速度 (如 1.5 MB/s)

  - 文件大小与进度 (如 120MB / 500MB)

  - 进度条视觉化展现 (0% - 100%)

  - 操作按钮 (根据状态显示：暂停、开始、删除)

### 3.5 数据持久化 (Data Persistence)

- **实时更新**：当任务状态发生改变（新建、暂停、完成、错误）时，必须将状态写入 SQLite 数据库。

- **进度节流**：下载过程中的进度更新每 2 秒批量写入数据库一次，避免频繁写盘。速度信息实时更新。

- **重启恢复**：软件启动时，需从 SQLite 读取所有历史任务。未完成的任务默认置为"已暂停"状态，已完成的任务进入"已完成"列表。

## 4. 性能与非功能需求 (Non-functional Requirements)

1. **线程隔离**：界面渲染（Main Thread）与文件下载、数据库读写（Worker Thread）必须严格物理隔离。UI 绝对不能因为大文件写入而发生卡顿或无响应。

2. **资源占用**：v1.0 单个任务下载时的内存占用峰值不得超过 50MB。

3. **刷新率**：前端界面的下载速度和进度条刷新频率应控制在 **1次/秒** 或 **2次/秒**，过高会导致 CPU 占用飙升且无实际意义。

---

## 5. 验收标准 (Acceptance Criteria)

当满足以下所有条件时，视为 v1.0 研发成功：

1. ✅ 能够成功编译并启动软件，无崩溃闪退。

2. ✅ 在"新建任务"中输入一个有效的大文件 HTTP 链接，能正常开始下载。

3. ✅ 下载过程中，进度条、已下载大小、当前速度能平滑更新。

4. ✅ 点击"暂停"，任务即刻停止；点击"继续"，能从上次断开的字节数接着下载（需服务器支持 Range 请求，若不支持则从头开始）。

5. ✅ 强行关闭软件并重新打开，之前的下载记录依然存在于列表中。

6. ✅ 文件下载到 100% 后，能在本地找到该文件，且文件能被正常打开且未损坏。

7. ✅ 添加相同 URL 的任务时，系统能正确提示"该 URL 正在下载中"。

8. ✅ 超过 3 个并发任务时，新任务会进入等待队列。

### 补充模块：环境规格说明 (Environment Specifications)

#### 1. 编译与开发环境 (Development & Build Environment)

为了保证团队开发的一致性以及底层库的完美兼容，v1.0 阶段严格限定以下开发工具链：

- **开发语言标准**：C++17 (选用 C++17 是为了在完美支持 Qt 6 的同时，保证各个编译器下最大的稳定性)。

- **核心 UI 框架**：Qt 6.5 LTS 或更高版本 (必须包含 `Qt Quick` 和 `Qt QML` 模块)。

- **构建系统**：CMake 3.20 或以上版本。

- **支持的编译器 (Windows 主力开发平台)**：

  - **首选**：MSVC 2022 (v143工具集) 64-bit。

  - **备选**：MinGW-w64 (GCC 11.2 或以上) 64-bit。

  - _(注：严禁使用 32 位编译器，以突破单文件 4GB 的内存寻址限制)_。

- **集成开发环境 (IDE)**：Qt Creator 11.0+ 或 Visual Studio 2022 (需安装 Qt Visual Studio Tools)。

- **本地数据库引擎**：SQLite 3 (直接使用 Qt SQL 模块自带的驱动，无需额外编译)。

#### 2. 运行与测试环境 (Execution & Testing Environment)

此环境定义了 XDown v1.0 承诺能够正常运行的用户电脑下限，也是 QA 测试阶段必须覆盖的基础环境。

- **操作系统要求 (OS)**：

  - **Windows**：Windows 10 (21H2 及以上版本) 或 Windows 11 (仅支持 64 位)。

  - _(注：v1.0 暂不将 macOS 和 Linux 列入强制测试环境，待后续迭代处理跨平台问题)_。

- **最低硬件要求 (Minimum Hardware)**：

  - **CPU**：双核处理器，1.5 GHz 及以上 (支持 x86-64 指令集)。

  - **内存 (RAM)**：4 GB。

  - **系统盘空间**：100 MB 可用空间 (仅用于安装软件及存放 SQLite 数据库文件)。

- **推荐硬件要求 (Recommended Hardware)**：

  - **CPU**：四核或以上处理器，2.0 GHz 及以上。

  - **内存 (RAM)**：8 GB 及以上 (有利于大文件下载时的内存缓冲区分配)。

  - **存储介质**：SSD 固态硬盘 (极其重要，机械硬盘在多线程高并发下载时极易造成 I/O 瓶颈，导致程序假死)。

- **网络环境**：

  - 需要标准 IPv4/IPv6 宽带网络接入。

  - 测试阶段需覆盖：正常局域网、存在代理/梯子的复杂网络环境、以及突发断网情况下的软件健壮性测试。
